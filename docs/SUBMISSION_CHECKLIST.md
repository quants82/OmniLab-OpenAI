# OpenAI Build Week submission checklist

Official deadline: July 21, 2026 at 5:00 PM Pacific Time. Treat the official Devpost rules and updates as the source of truth.

Evidence map: [`RULES_ALIGNMENT.md`](./RULES_ALIGNMENT.md).

## Eligibility and category

- [ ] Entrant is registered and eligible in their country/territory.
- [ ] If entering as a team or organization, the authorized representative is identified.
- [ ] Select exactly one track: **Education**.
- [ ] Confirm all third-party libraries, firmware images, media, and assets are authorized and license-compatible.
- [ ] Clearly disclose the pre-existing VatLi365 foundation and the new Build Week work.

## Working project

- [x] Public production URL available free of charge.
- [x] Public MIT-licensed repository.
- [x] Test account documented.
- [x] Six experiment pages and public firmware source.
- [x] GPT-5.6 Responses API integration visible in code and verified on production.
- [x] Local tests, production build, CI, automatic deployment, and health checks.
- [ ] Complete and record [`HARDWARE_ACCEPTANCE.md`](./HARDWARE_ACCEPTANCE.md) for all six experiments.
- [x] Add an explicitly labeled hardware-free flagship replay/demo path.
- [ ] Keep a working physical kit available in case judges request access.

## README and evidence

- [x] Setup and test instructions.
- [x] Sample/test access and live demo.
- [x] Specific explanation of Codex collaboration and key human decisions.

- [x] Pre-existing versus Build Week work documented with dated commits.
- [ ] Replace the README `/feedback` Session ID placeholder.

## Devpost text

- [x] Project name and concise tagline drafted.
- [x] Problem, audience, features, architecture, challenges, accomplishments, learnings, and next steps drafted.
- [x] Testing instructions and judge credentials drafted.
- [x] Claims distinguish demonstrated functionality from planned impact evaluation.
- [ ] Paste and proofread `DEVPOST_SUBMISSION.md` in the actual form.
- [ ] Verify every link in a signed-out browser.

## Video

- [ ] Record a working physical demonstration.
- [ ] Include English voiceover or English translation.
- [ ] Explain what was built, how Codex was used, and how GPT-5.6 is integrated.
- [ ] Show Codex briefly or show concrete Codex-created workflow evidence.
- [ ] Keep final duration at or below 3 minutes.
- [ ] Use only owned/authorized visuals, trademarks, and audio.
- [ ] Upload publicly to YouTube.
- [ ] Test playback while signed out.
- [ ] Add the public URL to README and Devpost.

## Final freeze

- [ ] Submit before the deadline; do not wait for the final hour.
- [ ] Save a local copy/screenshots of every submitted field.
- [ ] Confirm the project displays under **My projects** and is entered in the hackathon.
- [ ] Avoid substantive changes to submission materials after the submission period closes.
- [ ] Keep the website, judge account, repository, video, and hardware available through the judging period.
